# PAI v0.0.2

发布日期：2026-06-06

## 更新亮点

- ⚛️ **前端从 Vue 迁移到 React** — 全面替换为 React 18 + TypeScript，组件、入口、配置全部重写。
- 🔄 **集成 Tauri 更新器** — 新增应用内自动更新支持，用户可在桌面端直接检查和接收更新。

## 主要变更

### 前端框架迁移（Vue → React）

- 删除所有 Vue 组件：`App.vue`、`MainLayout.vue`、`HomeView.vue`、`SyncView.vue`。
- 重写为 React 函数组件（TSX）：`App.tsx`、`MainLayout.tsx`、`HomeView.tsx`、`SyncView.tsx`。
- 入口文件从 `main.ts` 迁移到 `main.tsx`，使用 React 18 的 `createRoot` API。
- 移除 Vue Router（`router/index.ts`），当前版本暂不引入客户端路由。
- 更新依赖：`package.json` 和 `package-lock.json` 切换到 React 生态（react、react-dom、@types/react 等）。

### Tauri 更新器集成

- 新增 `src/composables/useUpdater.tsx`，封装 Tauri 更新 API（检查更新、下载、安装）。
- 新增更新签名公钥 `src-tauri/pai-update-public.key`，用于验证更新包完整性。
- 更新 `tauri.conf.json` 和 `capabilities/default.json`，启用更新器权限。
- 更新 Tauri 生成模式文件（`gen/schemas/*`），支持更新器相关配置项。
- `Cargo.toml` 和 `Cargo.lock` 跟随 Tauri 更新器依赖调整。

### CI/CD 调整

- `.github/workflows/windows-release.yml` 更新，适配新的前端构建流程（React + Vite）和 Tauri 更新器签名。
- `.gitignore` 新增规则，排除更新私钥等敏感文件。

## 修复与优化

- 删除无用文件 `frontend/src/vite-env.d.ts`，保持项目清洁。
- TypeScript 配置（`tsconfig.json`）适配 React JSX 编译。
- Vite 配置（`vite.config.ts`）适配 React 插件和开发服务器设置。

## 技术与兼容性说明

- **前端框架**：从 Vue 3 迁移到 React 18（TypeScript + Vite）。
- **桌面框架**：Tauri 2.x（Rust 后端 + WebView 前端），当前仅配置 Windows 构建。
- **自动更新**：已集成 Tauri 更新器基础能力，后续发布将通过公钥签名分发包实现应用内更新提醒。
- **路由**：当前版本移除了 Vue Router，前端使用单视图结构；路由功能可能在后续版本以 React Router 形式回归。
- **最低兼容**：Windows 10+（当前仅提供 Windows 构建）。
